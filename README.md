## 🚀 サービス概要
「将来のビジョンはあるけれど、日々の忙しさに追われて、本当に大切なことに対しての行動ができない」
そんな課題を解決するために開発した、**未来志向のセルフマネジメントアプリ**です。
単なるタスク管理ではなく、長期目標（ビジョン）を常に意識しながら、今日のアクションを記録します。

## 🛠️ 使用技術
- **Backend**: Ruby 3.2.0 / Ruby on Rails 7.1
- **Frontend**: JavaScript / CSS (Vanilla)
- **Database**: PostgreSQL
- **Infrastructure**: Render
- **Testing**: RSpec (System Spec / Model Spec)

✅ テスト・品質管理
本アプリでは、継続的な開発と品質の安定を目的として、RSpecによる自動テストを導入しています。

System Spec (Capybara/Selenium):

ユーザーのログインから、カレンダー操作を通じたアクション投稿までの一連のフローを自動検証。

テストの堅牢性と信頼性の向上: UIのリファクタリングに伴い、アニメーションや要素の重なりでテストが不安定（Flaky）になる課題に直面しましたが、JavaScript（page.execute_script）を用いたイベントの直接発火や待機処理の最適化を行うことで、実行環境に左右されない堅牢なテストを実現しました。

Model Spec:

ユーザー登録時のバリデーションや、データ整合性のロジックを検証。

🛠️ 開発で直面した課題と解決策
「テスト駆動」による潜在的なバグの早期発見
単に「テストを通す」ことだけではなく、**「テストを利用して品質を担保する」**プロセスを重視しました。

課題の発見:
システムテストを安定化させた際、これまで表面化していなかった「保存失敗時の描画エラー（500エラー）」を特定しました。これは、例外処理時のコントローラー内でカレンダー描画に必要な変数が不足していたことが原因でした。

解決策と成果:
コントローラー側のロジックを共通化し、バリデーションエラー時にも適切な状態を維持できるよう修正しました。これにより、ユーザーが予期せぬエラー画面に遭遇することを未然に防ぎ、アプリケーション全体の堅牢性を高めることができました。

## 💡 開発背景
「成長したい」という強い思いがある一方で、日々の業務に追われ、本来大切にしたい価値観や長期的な目標を見失ってしまうという課題がありました。
この課題を解決するため、以下の2点を軸に開発しました。
1. **ビジョンの可視化**: 常に長期目標が目に入る状態を作り、思考の軸を固定する。
2. **行動への落とし込み**: 大きな目標を「今日の一歩（Action Plan）」に分解し、毎日記録することで、継続的な行動を促す。

## 📅 今後の実装予定（追加機能）

### 1. コンディション管理機能（睡眠・食事・運動の記録）
- **導入の狙い**: 
  長期的な目標達成には、一過性の努力ではなく「継続可能な心身のコンディション」が不可欠です。学習効率やモチベーションの変動を単なる根性論で片付けず、生活習慣という客観的データとの相関を可視化することで、科学的根拠に基づいた自己改善（セルフマネジメント）を可能にします。
- **期待される効果**: 
  パフォーマンスの最大化と、バーンアウト（燃え尽き）の未然防止。

### 2. データビジュアライゼーション（ダッシュボード機能）
- **導入の狙い**: 
  「今日の一歩」という抽象的な積み重ねを数値やグラフで視覚化することで、ユーザーに明確な達成感を提供します。蓄積されたデータが「自己効力感」を醸成し、モチベーションの維持を仕組み化することで、挫折しにくい環境をシステム側から構築します。
- **期待される効果**: 
  継続率の向上と、過去のログに基づいた精度の高い目標再設定。
